## 1. Общая информация о проекте

### 1.1. Название проекта
Локальный голосовой ассистент для CS2 с распознаванием речи, захватом экрана и LLM.

### 1.2. Цель проекта
Создать полностью локальную систему (без интернета и облачных сервисов), которая:
- По удержанию кнопки слушает микрофон, преобразует речь в текст (готовые промпты).
- Захватывает экран и парсит информацию (деньги, оружие, таймер, чат и т.д.).
- Обрабатывает данные через LLM с учётом контекста игры.
- Озвучивает ответ ассистента.
- Работает в реальном времени, оставляя максимум ресурсов для самой игры CS2.

Проект предназначен исключительно для личного использования и заточен под CS2.

### 1.3. Пример помощи в игре

- Я: Какое оружие купить?
- AI: На основе захвата экрана у тебя 2850 кредитов. Доступны: MP9 (1250) + броня, FAMAS (2050) + броня. Рекомендую MP9 + броня — идеально для твоей позиции на B и текущего стиля игры. После покупки останется 900 на следующий раунд.

## 2. Требования к системе

### 2.1. Функциональные требования

1. **STT (Speech-to-Text)**
   - Активация по удержанию кнопки (push-to-talk)
   - Преобразование речи в чистые предложения-промпты
   - Поддержка русского языка и сленга
   - Реал-тайм задержка 0.3–0.8 сек
   - Основная модель: faster-whisper large-v3-turbo (RealtimeSTT)
   - Выбор микрофона: по умолчанию + ручной индекс

2. **TTS (Text-to-Speech)**
   - Озвучка ответа LLM
   - Качественная Русская речь
   - Основная модель: **Qwen-TTS**
      - Качественная Русская речь
      - Возможность поставить любой голос
      - Тяжелая и медленная
      - Возможность работы CPU / CUDA
   - Запасная модель: **Silero TTS**
      - Качество похуже, но достаточно
      - Быстрая и легкая
      - CPU

3. **OCR (захват и парсинг экрана)**
   - Реал-тайм захват экрана
   - Распознавание текста и объектов (деньги, оружие, таймер, чат и т.д.)
   - Основная модель: **Florence-2-large**
      - Полноценная умная модель
      - Тяжелая
      - CPU / CUDA
   - Запасная модель: **EasyOCR**
      - Просто парсит текст в выделенных зонах
      - Легкая
      - CPU

4. **LLM (Large Language Model)**
   - Обработка промпта (STT + OCR + контекст игры)
   - Генерация полезных ответов с учётом экономики и истории раундов
   - Модель: Qwen 3 (4B/8B в зависимости от остатка памяти)
   - Управление через Ollama (unload после ответа)
   - Хранение базы данных в системах подобных AnythingLLM

5. **Интеграция и логика**
   - Полный пайплайн: Кнопка → STT → OCR → LLM + AnythingLLM → TTS
   - Lazy loading + выгрузка моделей
   - Система контекста CS2 (экономика раундов, стиль игры, история действий)
   - Мониторинг VRAM и FPS в реальном времени

### 2.2. Нефункциональные требования
- **Потребление памяти**: максимум 11–12 ГБ VRAM всего (игра + AI)
- **Задержки**: STT <1 сек, LLM <3 сек, TTS <2 сек
- **Работа**: 100% локально, без интернета
- **Совместимость**: Windows 11, Python 3.11, CUDA 12.8
- **Управление**: Config.yaml для всех моделей и параметров

## 3. Требования к тестированию
- Точность STT и OCR на русском в условиях CS2
- Качество и естественность TTS
- Полезность ответов LLM
- Стабильность FPS во время игры (цель — не ниже 200–250 FPS)
- Мониторинг VRAM и CPU в реальном времени

## 4. Риски и ограничения

- **Превышение VRAM** — основной риск (12 ГБ лимит).
- **Превышение контекста** — модель может забывать главную информацию.
- **Просадки FPS** из-за нагрузки на CPU.
- **Точность OCR** на мелком тексте CS2 (деньги, таймер).
- **Сложность контекста** — нужно хорошо хранить историю экономики и раундов + иметь БД с актуальной информацией.

**План минимизации рисков:**
- Всегда иметь быстрые fallback-модели
- Lazy loading + автоматический даунгрейд при приближении к лимиту VRAM
- Тестирование с запущенной CS2 на максимальных настройках
